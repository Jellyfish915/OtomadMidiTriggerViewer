# Otomad MIDI Trigger Viewer

![TypeScript](https://img.shields.io/badge/TypeScript-5.x-3178c6)
![Vite](https://img.shields.io/badge/Vite-7.x-646cff)
![PixiJS](https://img.shields.io/badge/PixiJS-8.x-e91e63)
![Preview only](https://img.shields.io/badge/export-not_yet-lightgrey)

**Otomad MIDI Trigger Viewer** は、MIDI のノート開始位置をトリガーにして、画像または動画素材をマスク内で表示・再生するローカルブラウザアプリです。音MADや映像編集の下作業として、ノートタイミングに合わせた反転、入り抜き、移動、拡縮、回転、方向ブラーの見え方を素早く確認できます。

現在の実装は **プレビュー専用** です。音声再生、動画書き出し、プロジェクトファイル保存はまだ含みません。

## Quick Start

Runtime: **Node.js 20.19+ または 22.12+**。

```bash
npm install
npm run dev
```

ブラウザで `http://127.0.0.1:5173/` を開きます。

```bash
npm run test
npm run build
```

## 使い方

1. `MIDIファイル` に `.mid` / `.midi` を読み込みます。
2. `画像/動画` に素材ファイルを読み込みます。
3. `BPM` と使用する `MIDIトラック` を設定します。
4. 必要に応じて、動画の `FPS`、開始フレーム、終了フレーム、時間制御カーブを設定します。
5. 表示領域の背景色、マスクサイズ、枠線、素材の座標・拡大率・縦横比率を調整します。
6. 入り抜きアニメーションを有効化し、拍数、パラメータ、ベジェ曲線を編集します。
7. 再生ボタンで、MIDIトリガーに同期したプレビューを確認します。

## Highlights

- **MIDIトリガー**: `@tonejs/midi` で note on を解析し、同一 tick の複数ノートは 1 トリガーとして扱います。
- **BPM基準の再計算**: MIDI内テンポではなく、入力BPMで tick / PPQ を秒へ変換します。
- **トラック選択**: 読み込んだMIDIのトラックを個別に有効化できます。
- **動画フレーム制御**: 開始フレーム、終了フレーム、FPS、ベジェ曲線による時間制御に対応しています。
- **固定マスク**: 素材はマスク内を cover で埋め、マスク自体には入り抜きアニメーションを適用しません。
- **素材補正**: 素材のX/Yオフセット、拡大率、横比率、縦比率を調整できます。
- **タイル表示**: マスク外へ素材が移動しても空白が出ないよう、ミラーまたは複製パターンで敷き詰めます。
- **入り抜き**: 入りと抜きを別々の効果リストとして管理します。次トリガーに向けて抜きが逆算開始され、新インスタンスの入りと同時に重なります。
- **複数効果の合成**: 座標移動、拡大縮小、回転、方向ブラー、反転を組み合わせられます。
- **設定保存**: BPM、FPS、マスク、背景色、曲線、入り抜き、ループ、トラック選択を localStorage に保存します。

## Preview Controls

- **再生 / 停止**: 現在位置からプレビューを開始・停止します。
- **先頭**: 先頭、またはA-Bループの開始拍へ戻します。
- **ループなし**: 通常再生します。
- **全体ループ**: MIDI全体の長さでループします。
- **A-Bループ**: 指定した開始拍・終了拍の範囲でループします。

## Animation

入りはトリガー開始時点から始まる登場アニメーションです。抜きは次のトリガー時点を終点として、指定拍数だけ前から逆算して始まる退場アニメーションです。

対応している効果:

- 座標移動: start / end のX・Yオフセットを補間します。
- 拡大縮小: start / end のスケールを補間します。
- 回転: start / end の角度を補間します。
- 方向ブラー: 指定角度に対してブラー強度を補間します。
- 反転: トリガーごとに左右、上下、または両方の反転を交互に切り替えます。

各補間は 0〜1 の2Dベジェグラフで編集できます。

## Development

```bash
npm install
npm run dev
```

開発サーバーは Vite で起動します。通常は `http://127.0.0.1:5173/` です。

```bash
npm run test
```

Vitest で MIDI解析、再生ロジック、easing、アニメーション合成のテストを実行します。

```bash
npm run build
```

TypeScript の型チェックと production build を実行します。成果物は `dist/` に出力されます。

## Project Structure

```text
src/
  animation.ts  入り抜き効果の補間と合成
  easing.ts     ベジェ曲線評価
  main.ts       UI、入力、localStorage、アプリ状態
  midi.ts       MIDI解析とトリガー生成
  playback.ts   再生位置とループ処理
  renderer.ts   PixiJS/WebGL プレビュー描画
  settings.ts   デフォルト設定と正規化
  types.ts      共有型定義
tests/
  *.test.ts     ロジック単体テスト
```

## Notes

- MIDIファイル本体と画像/動画素材本体は保存しません。再読み込み後は再選択が必要です。
- WebGL非対応環境ではプレビューを利用できません。
- 動画のFPSはファイルから自動推定せず、手入力の値を使います。
- 現時点ではブラウザ上のリアルタイムプレビューを目的としています。最終レンダリングや書き出しは外部編集ソフト側で行う想定です。
